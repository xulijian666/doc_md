# 概览

场景允许您与建模的宇宙进行交互，解锁“假如”问题的能力，以模拟不同的操作条件。要了解更多关于Palantir将模型与组织成果连接的高级方法，请参阅Ontology中的模型。

针对您的系统图配置，场景评估操作以及一个或多个建模输入，并计算输出值以反映您的数字孪生体的真实世界交互。Vertex的集成能力使您能够对多个交互集进行建模，并自动将这些交互串联在一起，将一个模型的输出作为另一个模型的输入。这样您就可以理解和交互多个系统中的多个过程，以了解拟议更改的端到端影响。

## Foundry中的业务逻辑和模型

为了模拟操作条件，您首先需要定义这些条件、它们的Ontology关系以及预期行为。您可以通过创建建模目标来实现这一点，在Foundry中撰写、评估和部署模型。

将模型视为任务，它们接受预定义的输入集并返回一组计算输出。模型版本指定了输入和输出参数，并且可以与您的系统图中看到的Ontology对象一起配置。这使您能够将建模概念与数字孪生体紧密对齐，以提供动态系统交互。了解更多关于Foundry中的机器学习和建模。。

一旦发布，Foundry建模目标将在Vertex中可供配置，您可以交互运行场景以了解潜在操作条件的影响。

Foundry中的函数使代码作者能够编写可以在操作环境中快速执行的逻辑，例如旨在支持决策过程的仪表盘和应用程序。一旦发布，函数也可以在Vertex中用于支持动态模拟案例研究。了解更多关于函数的信息。

## 场景的时间序列

为了理解和与您的系统随时间的更改进行交互，关键是将您的测量值塑造成时间序列，以便将这些值配置为模型的输入，从而生成用于比较的计算时间序列输出。这将允许您监控当前状态，查看历史趋势，并通过模拟覆盖建模条件来预测未来的更改。

您可以在时间序列文档中找到关于时间序列设置的更多信息。请联系您的Palantir代表以获得时间序列的进一步帮助。
