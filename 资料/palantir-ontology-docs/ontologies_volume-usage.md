# Ontology 占用量

Foundry 的 Ontology 是一个快速访问的存储层，使您能够将对象定义绑定到交互数据，以便进行查询、链接、场景和操作。Foundry Ontology 中的数据被索引，以便快速访问，并允许多个编辑者同时安全地进行编辑。Ontology 占用量是已索引对象集及其相互之间链接的总大小的度量。Ontology 占用量是一个平均指标，其单位为 GB，用于瞬时读取，但在一个月的时间内测量时单位为 GB-月。

## 测量 Ontology 占用量

Ontology 占用量通过测量支持对象类型的索引大小来记录。每个对象类型都有一定数量的对象和每个对象的属性数量。每个属性可以是任意大小。索引的总大小是通过对该对象类型的每个对象的每个已索引属性的大小进行求和来计算的。

需要注意的是，Ontology 占用量可能大于数据集占用量，因为 Ontology 数据无法压缩，并且 Ontology 索引需要额外的存储来促进更快的查询。

每小时，Foundry 平台会记录每个对象的 Ontology 占用量的测量值。在随时间测量 Ontology 占用量时，所有的每小时测量值在给定时间段内取平均值。平均值在一个日历月内产生GB-月单位。

## 调查 Ontology 占用量使用情况

对象在 Ontology Manager 中进行管理，这是所有对象管理和监控的枢纽。Ontology Manager 允许用户配置哪些数据集应成为对象，哪些类型的属性附加到这些对象上，以及在对象类型之间定义哪些链接集。

对象及其对应的链接类型的总 Ontology 占用量在资源管理应用程序中列出。

## 影响 Ontology 占用量的因素

Ontology 占用量有效地衡量了本体中的所有对象的大小，包括它们的属性和链接。以下两个因素是总占用量的主要驱动因素。

- 每个对象类型的对象数量和大小当从数据集中创建一个 Ontology 对象类型时，每行数据集都会被索引为一个对象。因此，该数据集中的行数与相应对象类型中的对象数量是 1:1 相关的。此外，具有更多列或包含更多数据（例如，自由文本字段）的数据集可以生成更大的单个对象，因为每个列都变成了一个属性。
- 使用合并表时对象之间的链接数量在多对多关系中，Ontology 需要定义一个合并表，以基于它们的主键定义所有对象之间的链接。这些表与 Ontology 中的对象一起被索引，并使用 Ontology 占用量。通常，查找表的每条记录大小是恒定的，并且随着所定义链接数量的增加，其占用量线性增长。

## 管理 Ontology 占用量

Ontology 旨在成为操作使用和查询的快速访问后端。在一般情况下，Ontology 最好与从更大的数据资产中合成的高度精炼数据一起使用。因此，Ontology 的占用量应小于 Foundry 变换框架中原始和中间数据的总大小。

要管理 Ontology 占用量使用，请注意 Ontology 中定义的对象类型数量，以及每个对象类型的对象数量和每个对象的属性数量。总体而言，管理 Ontology 占用量使用的最佳方法是理解并有意识地管理对象数量、属性计数和属性大小。
