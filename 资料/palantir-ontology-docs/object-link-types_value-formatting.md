支持的值格式化

注意：以下翻译的准确性尚未经过验证。这是使用AIP ↗从原始英文文本进行的机器翻译。

值格式化是指对属性的值应用一个特殊的格式器，将原始值变换为更易读的版本。在下图中，左侧（之前）显示了weight和value列没有任何格式化。右侧（之后）在weight列应用了单位（“kg”），而value列则以更紧凑的形式显示，并附有货币符号（“$100K”）。这些都是数字格式化的例子。Ontology还支持日期和时间格式化，以及用户ID格式化和资源RID格式化。

## 支持的值格式化

| 类型 | 描述 |
| --- | --- |
| 数字格式化 | 添加货币/单位/前缀，各种类型的符号（紧凑，科学）和百分比。查看数字格式化部分以获取更多详情。 |
| 日期和时间格式化 | 以特定格式和特定时区呈现时间戳和日期。 |
| Foundry ID格式化 | 将Foundry ID显示为用户的名字和姓氏或组名。 |
| 资源RID格式化 | 将Foundry资源ID（RID）显示为图标和资源名称，并提供可点击的链接以导航到该资源。 |

## 添加值格式化

在属性编辑器中：

1. 选择您想要添加值格式化的属性。
2. 在属性窗格的右侧面板中，您将看到根据属性的基本类型（值格式化，数字格式化，日期和时间格式化等）显示的格式类型。开启格式化。
3. 如下所述，数字格式化和日期和时间格式化有附加的格式化选项：数字格式化选项日期和时间格式化选项
4. 当您选择可用的格式化选项时，您将看到应用新格式化后属性值的预览。

### 数字格式化选项

| 名称 | 描述 | 用法 |
| --- | --- | --- |
| 数字格式化 | 开关。 | 切换此选项以移除/添加数字格式化。 |
| 基本类型 | 包含可用的各种格式化类型（货币、单位、百分比、前缀/后缀、固定值）以及每种类型的示例和描述。 | 如果Capacity in Pounds有一个相关联的单位，请从此下拉菜单中选择“单位”。 |
| 使用分组 | 添加符合区域设置的逗号分隔符。 | 开启此选项以从123456变为123,456。 |
| 符号 | 包含紧凑/科学和工程符号。 | 选择紧凑以近似值，如123K。 |
| 预览结果 | 查看和测试数字格式化。 | 在输入框中添加任何类似于您期望在属性值中看到的数字，以预览格式化效果。 |

### 日期和时间格式化选项

| 名称 | 描述 | 示例 |
| --- | --- | --- |
| 日期 | 只有日期（无时间） | Wed, Jul 22, 2020 |
| 日期和时间（长格式） | 日期和时间，长格式 | Wed, July 22, 2020, 1:00:00 PM |
| 日期和时间（短格式） | 日期和时间，短格式 | Jul 22, 2020, 1:00 PM |
| ISO即时 | 日期和时间（ISO 8601格式） | 2020-07-22T13:00:00.000Z |
| 相对于现在 | 相对于当前的日期 | 8 minutes ago |
| 时间 | 只有时间（无日期） | 1:00 pm |

当格式化为相对于现在时，应用程序将仅在相对术语中格式化到24小时之前。之后，它将以带有星期几的**日期和时间（短格式）**形式呈现：Wed, Jul 22, 2020, 1:00 PM。

#### 时区

如果您正在格式化时间戳，可以指定在哪个时区呈现时间戳，既可以是您输入的静态时区，也可以是应用程序用户的当前时区。在输入静态时区时，可以通过输入UTC偏移量或区域名称来搜索时区。

### 用户ID格式化

值格式化可以应用于Foundry/Multipass用户ID或组ID的字符串，并通过选择Multipass用户名选项将其转换为显示用户的名字和姓氏或组名。此值格式化选项通常用于您创建了一个操作来编辑一个属性并在属性字段之一中存储用户ID或组ID时。在后备数据中，此信息将存储为用户的Foundry用户ID或组ID，而值格式化可以应用于呈现用户的名字或组名而不是ID。

## 常见问题

### 这会与现有的类型类一起工作吗？

在UI中，值格式化优先于现有的类型类。如果您同时配置了两者，将显示值格式化。不过，您可以在一个属性上使用值格式化，在另一个属性上使用类型类。

### 这会与Object Views中的可编辑属性一起工作吗？

值格式化支持配置为行内编辑的属性。具有遗留hubble:editable类型类的属性将禁用值格式化。
