## 1. 监控体系

### 1.1 监控层次

| 层次 | 监控工具 | 监控内容 |
|------|----------|----------|
| 基础设施 | Prometheus + Node Exporter | CPU/内存/磁盘/网络 |
| 容器层 | Prometheus + cAdvisor | Pod资源/容器状态 |
| 应用层 | Spring Boot Actuator + Micrometer | JVM/接口/线程池 |
| 业务层 | 自研业务监控平台 | 投保量/成功率/转化率 |
| 链路追踪 | SkyWalking | 调用链/慢查询/异常 |
| 日志 | ELK Stack | 日志采集/分析/告警 |

### 1.2 监控架构

```
┌─────────────────────────────────────────────────────────┐
│                      数据采集层                           │
│  Node Exporter │ cAdvisor │ Micrometer │ Filebeat       │
└────────┬────────┴────┬─────┴─────┬──────┴────┬──────────┘
         │             │           │           │
┌────────▼─────────────▼───────────▼───────────▼──────────┐
│                      数据存储层                           │
│  Prometheus │ InfluxDB │ Elasticsearch │ SkyWalking      │
└────────┬────────────┬────────────┬────────────┬─────────┘
         │            │            │            │
┌────────▼────────────▼────────────▼────────────▼─────────┐
│                      展示与告警层                         │
│  Grafana │ 告警管理平台 │ 钉钉/短信/电话                   │
└─────────────────────────────────────────────────────────┘
```

## 2. 告警规则

### 2.1 基础设施告警

| 告警名称 | 条件 | 级别 | 持续时间 | 通知方式 |
|----------|------|------|----------|----------|
| CPU使用率高 | CPU > 80% | P2 | 5分钟 | 钉钉 |
| CPU使用率极高 | CPU > 95% | P1 | 2分钟 | 短信+钉钉 |
| 内存使用率高 | Memory > 85% | P2 | 5分钟 | 钉钉 |
| 内存使用率极高 | Memory > 95% | P1 | 2分钟 | 短信+钉钉 |
| 磁盘使用率高 | Disk > 80% | P2 | 5分钟 | 钉钉 |
| 磁盘使用率极高 | Disk > 90% | P1 | 即时 | 短信+钉钉 |
| 节点宕机 | Node Up = 0 | P0 | 即时 | 电话+短信+钉钉 |

### 2.2 应用告警

| 告警名称 | 条件 | 级别 | 持续时间 | 通知方式 |
|----------|------|------|----------|----------|
| Pod重启 | Pod Restart > 3次/5分钟 | P1 | 即时 | 短信+钉钉 |
| Pod未就绪 | Pod Ready = False | P1 | 2分钟 | 短信+钉钉 |
| 接口响应慢 | P99 > 5秒 | P2 | 5分钟 | 钉钉 |
| 接口错误率高 | Error Rate > 1% | P1 | 3分钟 | 短信+钉钉 |
| 接口错误率极高 | Error Rate > 5% | P0 | 1分钟 | 电话+短信+钉钉 |
| JVM堆内存高 | Heap Usage > 80% | P2 | 5分钟 | 钉钉 |
| Full GC频繁 | Full GC > 5次/分钟 | P1 | 3分钟 | 短信+钉钉 |
| 线程池满 | Active Threads > 90% | P1 | 2分钟 | 短信+钉钉 |
| 连接池满 | DB Connections > 90% | P1 | 2分钟 | 短信+钉钉 |

### 2.3 中间件告警

| 告警名称 | 条件 | 级别 | 持续时间 | 通知方式 |
|----------|------|------|----------|----------|
| MySQL主从延迟 | Seconds_Behind > 10s | P1 | 3分钟 | 短信+钉钉 |
| MySQL连接数高 | Threads_connected > 80% | P2 | 5分钟 | 钉钉 |
| MySQL慢SQL | 慢SQL > 100条/分钟 | P2 | 5分钟 | 钉钉 |
| Redis内存高 | Used Memory > 85% | P2 | 5分钟 | 钉钉 |
| Redis连接数高 | Connected Clients > 1000 | P2 | 5分钟 | 钉钉 |
| MQ消息堆积 | Consumer Lag > 10000 | P1 | 5分钟 | 短信+钉钉 |
| MQ消费失败 | Consume Fail > 100条/分钟 | P1 | 3分钟 | 短信+钉钉 |
| ES集群健康 | Cluster Health = Red | P0 | 即时 | 电话+短信+钉钉 |
| ES集群健康 | Cluster Health = Yellow | P2 | 5分钟 | 钉钉 |

### 2.4 业务告警

| 告警名称 | 条件 | 级别 | 持续时间 | 通知方式 |
|----------|------|------|----------|----------|
| 投保成功率低 | 成功率 < 95% | P1 | 5分钟 | 短信+钉钉 |
| 投保成功率极低 | 成功率 < 90% | P0 | 2分钟 | 电话+短信+钉钉 |
| 支付成功率低 | 成功率 < 98% | P1 | 5分钟 | 短信+钉钉 |
| 投保量突增 | 投保量 > 日均×2 | P2 | 即时 | 钉钉 |
| 投保量骤降 | 投保量 < 日均×0.3 | P1 | 10分钟 | 短信+钉钉 |
| 大额投保 | 单笔保费 > 50万 | P3 | 即时 | 邮件 |
| 退保率突增 | 退保率 > 日均×2 | P2 | 1小时 | 钉钉 |

## 3. Grafana面板

### 3.1 面板清单

| 面板名称 | 内容 | 访问权限 |
|----------|------|----------|
| 系统总览 | 全局资源使用率+服务状态 | 运维+开发 |
| 应用监控 | 各服务JVM+接口指标 | 开发 |
| 业务大盘 | 投保量/理赔量/保费收入 | 全员 |
| 数据库监控 | MySQL/Redis/TiDB | DBA+运维 |
| 中间件监控 | MQ/ES/Nacos | 运维 |
| 链路追踪 | 慢查询+异常链路 | 开发 |

### 3.2 关键指标

**业务大盘核心指标：**
- 实时投保量（分钟级）
- 今日投保成功率
- 今日保费收入
- 实时在线用户数
- 核心接口P99响应时间

## 4. 告警通知配置

### 4.1 通知渠道

| 渠道 | 配置 | 适用场景 |
|------|------|----------|
| 钉钉机器人 | Webhook URL | 所有级别告警 |
| 短信 | 阿里云SMS | P0/P1级别告警 |
| 电话 | 阿里云语音 | P0级别告警 |
| 邮件 | SMTP | P3级别告警+日报 |

### 4.2 告警升级

| 时间 | 未处理 | 升级动作 |
|------|--------|----------|
| 0分钟 | 告警触发 | 钉钉通知值班人 |
| 5分钟 | P0未响应 | 短信通知值班人 |
| 10分钟 | P0未响应 | 电话通知值班人+主管 |
| 30分钟 | P0未解决 | 通知技术负责人 |
| 60分钟 | P0未解决 | 通知CTO |

### 4.3 告警抑制

| 场景 | 抑制规则 |
|------|----------|
| 维护窗口 | 暂停所有告警 |
| 已知故障 | 抑制关联告警 |
| 告警风暴 | 5分钟内同类告警合并 |
