## 1. 项目背景

公司健康险业务年增长率超过 40%，理赔案件量日均突破 8000 笔。现有理赔系统存在以下痛点：

- 理赔材料审核依赖人工，平均处理时效 5.2 个工作日
- 医疗数据对接不完整，需要客户反复补交材料
- 欺诈识别能力不足，年欺诈损失率约 2.3%

本项目旨在建设智能化理赔平台，实现小额案件自动审核、大额案件智能辅助决策。

## 2. 业务目标

| 目标编号 | 目标描述 | 量化指标 |
|----------|----------|----------|
| BG-01 | 缩短理赔时效 | 小额案件（≤5000元）≤ 1个工作日 |
| BG-02 | 提升自动化率 | 自动审核通过率 ≥ 60% |
| BG-03 | 降低欺诈损失 | 欺诈损失率 ≤ 1.0% |
| BG-04 | 提升客户满意度 | 理赔NPS评分 ≥ 75 |

## 3. 理赔类型

| 类型 | 描述 | 理赔金额上限 | 审核方式 |
|------|------|-------------|----------|
| 住院医疗 | 因疾病或意外住院产生的医疗费用 | 50万 | ≤5000自动，>5000人工 |
| 门诊医疗 | 门诊挂号、检查、药品费用 | 5万 | ≤2000自动，>2000人工 |
| 重大疾病 | 确诊合同约定的重大疾病 | 100万 | 全部人工审核 |
| 意外伤残 | 因意外导致的伤残 | 按伤残等级 | 全部人工审核 |
| 住院津贴 | 按住院天数给付 | 300元/天 | ≤15天自动，>15天人工 |

## 4. 核心业务流程

### 4.1 理赔报案

#### FR-001 多渠道报案

| 报案渠道 | 功能要求 | 响应时效 |
|----------|----------|----------|
| APP自助报案 | 引导式填写+OCR识别材料 | 即时受理 |
| 电话报案 | IVR引导+人工坐席 | ≤ 3分钟 |
| 医院直赔 | HCP系统自动触发 | 自动受理 |
| 线下柜台 | 柜员代为录入 | 即时受理 |

#### FR-002 报案信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 保单号 | 字符串 | 是 | 关联保单信息自动带出 |
| 出险人 | 字符串 | 是 | 从保单被保险人列表选择 |
| 出险日期 | 日期 | 是 | 不得晚于当前日期，不得早于保险起期 |
| 出险原因 | 枚举 | 是 | 疾病/意外/其他 |
| 就诊医院 | 字符串 | 是 | 关联医院库 |
| 诊断结果 | 字符串 | 是 | ICD-10编码 |
| 预估金额 | 数值 | 否 | 参考值 |

### 4.2 材料收集

#### FR-003 必要材料清单

| 理赔类型 | 必要材料 |
|----------|----------|
| 住院医疗 | 身份证、银行卡、住院病历、费用明细、发票原件 |
| 门诊医疗 | 身份证、银行卡、门诊病历、处方单、发票 |
| 重大疾病 | 身份证、银行卡、病理报告、诊断证明、住院病历 |
| 意外伤残 | 身份证、银行卡、伤残鉴定报告、事故证明 |

#### FR-004 OCR材料识别

- 支持识别发票、病历、诊断证明等 12 种材料类型
- 识别准确率 ≥ 95%
- 自动提取关键字段（金额、日期、诊断编码等）
- 识别结果人工确认后方可进入审核

### 4.3 理赔审核

#### FR-005 自动审核规则

```
自动审核通过条件（全部满足）：
1. 理赔金额 ≤ 阈值（住院5000/门诊2000/津贴15天）
2. 材料完整且OCR识别一致
3. 保单状态有效
4. 非免责条款范围
5. 风险评分 < 60分
6. 非重复报案
7. 就诊医院在白名单内
```

#### FR-006 风险评分模型

| 风险因子 | 权重 | 数据来源 |
|----------|------|----------|
| 报案频率 | 20% | 历史理赔记录 |
| 就诊医院等级 | 15% | 医院库 |
| 诊断与险种匹配度 | 25% | 规则引擎 |
| 投保后短期出险 | 20% | 保单数据 |
| 地域风险系数 | 10% | 风控平台 |
| 关联人风险 | 10% | 知识图谱 |

#### FR-007 人工审核

- 审核界面左侧展示材料影像，右侧展示OCR提取结果
- 支持在线查看保单信息、历史理赔记录、风险评分详情
- 审核结论：通过/部分通过/拒赔/补充材料
- 拒赔需选择拒赔理由（关联免责条款）

### 4.4 理赔支付

#### FR-008 赔款计算

```
赔款金额 = (合理医疗费用 - 免赔额) × 赔付比例 - 已从其他途径补偿金额
```

| 产品类型 | 免赔额 | 赔付比例 |
|----------|--------|----------|
| 百万医疗 | 10000元/年 | 100%（社保后）/ 60%（未经社保） |
| 小额医疗 | 0元 | 80% |
| 重疾险 | 0元 | 100%（定额给付） |
| 住院津贴 | 0天免赔 | 100%（按天数×日额） |

#### FR-009 支付方式

- 赔款直接支付至被保险人本人银行卡
- 支持对公账户（企业团险）
- 支付时效：审核通过后 ≤ 2个工作日

### 4.5 理赔争议处理

#### FR-010 申诉流程

- 客户可在拒赔后 15 个工作日内提出申诉
- 申诉由高级核赔师复审
- 复审结果为最终结论
- 支持外部调解（保险行业协会）对接

## 5. 接口需求

| 接口 | 方向 | 协议 | 用途 |
|------|------|------|------|
| 社保数据接口 | 入站 | HTTPS | 获取社保报销数据 |
| 医院HIS接口 | 入站 | HL7 FHIR | 获取就诊/费用数据 |
| 银联支付接口 | 出站 | HTTPS | 赔款支付 |
| 征信查询接口 | 出站 | HTTPS | 风控信息查询 |
| 反欺诈平台接口 | 出站 | gRPC | 风险评分 |

## 6. 非功能需求

| 指标 | 要求 |
|------|------|
| 报案接口响应时间 | ≤ 2s |
| OCR识别耗时 | ≤ 10s/页 |
| 自动审核决策时间 | ≤ 30s |
| 日处理案件量 | ≥ 15000笔 |
| 数据保留期限 | ≥ 10年 |
