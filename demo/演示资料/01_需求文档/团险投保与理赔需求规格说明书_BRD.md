## 1. 项目背景

团体保险业务是公司重要的B端业务线，当前面临以下挑战：

- 企业客户自助服务能力不足，大量操作依赖客户经理
- 团险方案灵活度高，现有系统配置能力有限
- 批量增减员操作效率低，影响企业HR体验

## 2. 业务目标

| 目标编号 | 目标描述 | 量化指标 |
|----------|----------|----------|
| BG-01 | 企业自助率 | 企业HR自助操作占比 ≥ 70% |
| BG-02 | 增减员时效 | 批量增减员处理 ≤ 30分钟 |
| BG-03 | 方案灵活度 | 支持 ≥ 20种方案组合模板 |
| BG-04 | 团险理赔时效 | 团险理赔 ≤ 3个工作日 |

## 3. 业务角色

| 角色 | 描述 | 权限 |
|------|------|------|
| 企业HR | 企业人力资源管理员 | 增减员、方案变更、理赔申请、账单管理 |
| 客户经理 | 公司团险业务人员 | 方案设计、报价、企业服务 |
| 团险核保人 | 团险核保岗位 | 方案审核、特殊风险评估 |
| 财务人员 | 保费收取与对账 | 账单生成、对账、发票管理 |

| 模板编号 | 模板名称 | 适用场景 | 包含险种 |
|----------|----------|----------|----------|
| T-001 | 基础员工福利 | 小微企业 | 团体意外+团体医疗 |
| T-002 | 标准员工福利 | 中型企业 | 团体意外+团体医疗+团体重疾 |

## 5. 增减员管理

### 5.1 增员

#### FR-001 单个增员

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | string | 是 | |
| 证件号 | string | 是 | 身份证/护照 |
| 出生日期 | date | 是 | 用于费率计算 |
| 性别 | enum | 是 | |
| 职位 | string | 否 | |
| 入职日期 | date | 是 | |
| 生效日期 | date | 是 | 默认次月1日 |

#### FR-002 批量增员

- 支持Excel模板上传，单次 ≤ 500人
- 自动校验：证件号格式、重复检查、年龄限制
- 校验失败的记录标红，支持修正后重新提交
- 批量处理结果异步通知

### 5.2 减员

- 支持单个/批量减员
- 减员生效日期可选：当日/当月末/指定日期
- 减员后自动计算退费金额（按天折算）

## 6. 团险理赔

### 6.1 理赔流程

```
员工报案（APP/HR代报）
    │
    ▼
材料收集（OCR识别+在线补充）
    │
    ▼
初审（系统自动规则校验）
    │
    ├── 小额案件（≤2000元）──▶ 自动审核 ──▶ 支付
    │
    └── 大额案件 ──▶ 人工审核 ──▶ 支付/拒赔
```

### 6.2 团险特殊规则

| 规则 | 说明 |
|------|------|
| 在保验证 | 理赔时验证员工是否仍在保 |
| 等待期校验 | 新入职员工需过等待期 |
| 团体限额 | 部分方案设有团体年度限额 |
| 既往症排除 | 新加保员工的既往症可设等待期 |

## 7. 账单与对账

### 7.1 保费账单

| 账单类型 | 生成周期 | 说明 |
|----------|----------|------|
| 月度账单 | 每月1日 | 按当月在保人数计算 |
| 增减员调整单 | 实时 | 增减员引起的保费差额 |
| 年度结算单 | 每年1月 | 上年度多退少补 |

### 7.2 对账流程

- 每月5日前生成上月账单
- 企业HR确认账单（7个工作日内）
- 支持在线支付/银行转账
- 确认后自动开具增值税专用发票
