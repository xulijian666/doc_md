## 1. 项目背景

公司计划拓展互联网保险分销渠道，需对接主流流量平台（支付宝、微信、京东、抖音等），实现产品上架、投保、出单、理赔全流程线上化。

## 2. 对接渠道

| 渠道 | 对接模式 | 预计日单量 | 优先级 |
|------|----------|-----------|--------|
| 支付宝·蚂蚁保 | API对接 | 5000+ | P0 |
| 微信·微保 | API对接 | 3000+ | P0 |
| 京东·京东保险 | API对接 | 2000+ | P1 |
| 抖音·保险服务 | H5嵌入 | 1500+ | P1 |
| 水滴保 | API对接 | 1000+ | P2 |

## 3. 统一网关设计

### 3.1 网关架构

```
渠道方 ──▶ 统一接入网关 ──▶ 协议转换层 ──▶ 业务服务层
              │                    │                │
              ▼                    ▼                ▼
         鉴权/限流           报文格式转换      核心业务逻辑
         签名验证           字段映射          数据持久化
         流量控制           参数校验          结果返回
```

### 3.2 接口规范

| 规范项 | 要求 |
|--------|------|
| 协议 | HTTPS + RESTful |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 签名算法 | RSA2（SHA256WithRSA） |
| 时间戳 | 请求时间与服务器时间偏差 ≤ 5分钟 |
| 幂等性 | 关键接口支持幂等（投保、支付） |

## 4. 核心接口定义

### 4.1 产品查询接口

```
POST /api/v1/channel/product/query
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| channelCode | string | 是 | 渠道编码 |
| productCode | string | 否 | 产品编码，空则查询全部 |
| category | string | 否 | 产品类别：CAR/HEALTH/LIFE/PROPERTY |

**响应参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| productList | array | 产品列表 |
| productList[].productCode | string | 产品编码 |
| productList[].productName | string | 产品名称 |
| productList[].premium | object | 费率信息 |
| productList[].coverage | array | 保障范围 |

### 4.2 投保报价接口

```
POST /api/v1/channel/insurance/quote
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| channelCode | string | 是 | 渠道编码 |
| productCode | string | 是 | 产品编码 |
| applicant | object | 是 | 投保人信息 |
| insured | array | 是 | 被保险人列表 |
| effectiveDate | string | 是 | 生效日期 |

### 4.3 投保确认接口

```
POST /api/v1/channel/insurance/submit
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| quoteNo | string | 是 | 报价单号 |
| paymentInfo | object | 是 | 支付信息 |
| health告知 | array | 否 | 健康告知（健康险必填） |

### 4.4 保单查询接口

```
GET /api/v1/channel/policy/{policyNo}
```

### 4.5 理赔报案接口

```
POST /api/v1/channel/claim/report
```

### 4.6 退保接口

```
POST /api/v1/channel/insurance/surrender
```

## 5. 数据映射

### 5.1 险种代码映射

| 内部编码 | 支付宝编码 | 微信编码 | 产品名称 |
|----------|-----------|----------|----------|
| CAR-COMP | ALI_CAR_C | WX_CAR_C | 交强险 |
| CAR-BIZ | ALI_CAR_B | WX_CAR_B | 商业车险 |
| HL-MED | ALI_HL_M | WX_HL_M | 百万医疗 |
| HL-CRI | ALI_HL_C | WX_HL_C | 重疾险 |
| LF-WHL | ALI_LF_W | WX_LF_W | 终身寿险 |

### 5.2 状态映射

| 内部状态 | 外部状态 | 说明 |
|----------|----------|------|
| PENDING | INIT | 待处理 |
| UNDERWRITING | PROCESSING | 核保中 |
| APPROVED | SUCCESS | 核保通过 |
| REJECTED | FAILED | 核保拒绝 |
| EFFECTIVE | ACTIVE | 已生效 |
| SURRENDERED | CANCELLED | 已退保 |

## 6. 安全要求

| 安全项 | 要求 |
|--------|------|
| 接口鉴权 | OAuth 2.0 + 签名验证 |
| IP白名单 | 渠道方固定IP段 |
| 数据加密 | 敏感字段AES加密传输 |
| 请求限流 | 单渠道 QPS ≤ 1000 |
| 日志审计 | 全量请求/响应日志，保留3年 |

## 7. 异常处理

| 错误码 | 含义 | 处理方式 |
|--------|------|----------|
| 10001 | 参数校验失败 | 返回具体字段错误信息 |
| 10002 | 签名验证失败 | 拒绝请求，记录告警 |
| 10003 | 产品不存在 | 返回产品下架提示 |
| 10004 | 核保失败 | 返回核保拒绝原因 |
| 10005 | 支付超时 | 建议重试，保留报价 |
| 20001 | 系统异常 | 返回通用错误，记录日志 |
