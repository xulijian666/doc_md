## 1. 项目背景

随着公司车险业务规模持续增长，现有投保系统在高并发场景下出现响应超时、核保规则引擎性能瓶颈等问题。本项目旨在重构车险投保核心流程，支持日均 50 万笔投保请求，平均响应时间控制在 500ms 以内。

## 2. 业务目标

| 目标编号 | 目标描述 | 量化指标 |
|----------|----------|----------|
| BG-01 | 提升投保转化率 | 线上投保完成率从 62% 提升至 85% |
| BG-02 | 缩短出单时效 | 从提交到出单平均耗时 ≤ 3 分钟 |
| BG-03 | 降低人工核保比例 | 自动核保通过率 ≥ 90% |
| BG-04 | 支持多渠道接入 | 统一 API 对接 15+ 渠道方 |

## 3. 业务范围

### 3.1 在范围内

- 交强险投保流程
- 商业车险投保流程（车损险、第三者责任险、车上人员责任险等）
- 费率计算与优惠因子应用
- 核保规则引擎（自动核保 + 人工核保队列）
- 电子保单生成与推送
- 投保单状态管理（草稿→待核保→核保通过→已出单→已生效）

### 3.2 不在范围内

- 理赔流程（见 REQ-CLM-2026-001）
- 续保提醒与自动续保（见 REQ-RNW-2026-001）
- 渠道佣金结算（见 REQ-CMS-2026-001）

## 4. 用户角色

| 角色 | 描述 | 权限 |
|------|------|------|
| C端客户 | 通过APP/小程序自助投保 | 查看产品、填写信息、支付、查看保单 |
| 代理人 | 代理渠道的业务人员 | 代客投保、查看名下保单、佣金查询 |
| 核保员 | 人工核保岗位 | 查看待核保任务、审核、批注、通过/驳回 |
| 系统管理员 | 系统配置与维护 | 规则配置、费率维护、权限管理 |

## 5. 功能需求

### 5.1 投保信息录入

#### FR-001 车辆信息录入

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 车牌号 | 字符串 | 是 | 标准车牌格式，支持新能源车牌 |
| 车架号(VIN) | 字符串 | 是 | 17位，校验位算法验证 |
| 发动机号 | 字符串 | 是 | 与行驶证一致 |
| 品牌型号 | 字符串 | 是 | 关联车型库，支持模糊搜索 |
| 初次登记日期 | 日期 | 是 | 不得晚于当前日期 |
| 使用性质 | 枚举 | 是 | 家庭自用/企业非营运/营运 |
| 座位数 | 整数 | 是 | 1-55 |

#### FR-002 投保人/被保险人信息

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 姓名 | 字符串 | 是 | 2-20个汉字 |
| 证件类型 | 枚举 | 是 | 身份证/营业执照/护照 |
| 证件号码 | 字符串 | 是 | 根据证件类型匹配格式 |
| 手机号 | 字符串 | 是 | 11位，短信验证码校验 |
| 联系地址 | 字符串 | 是 | 省市区+详细地址 |

#### FR-003 险种选择

- 支持交强险单独投保
- 支持商业险多险种组合投保
- 险种之间依赖关系自动处理（如投保车损险后才可投保涉水险）
- 各险种保额可选范围动态加载

### 5.2 费率计算

#### FR-004 保费计算逻辑

```
最终保费 = 基准纯保费 × NCD系数 × 交通违法系数 × 自主定价系数 × 渠道系数
```

| 系数 | 数据来源 | 取值范围 |
|------|----------|----------|
| NCD系数 | 无赔款优待平台 | 0.6 ~ 2.0 |
| 交通违法系数 | 交管数据平台 | 0.9 ~ 1.5 |
| 自主定价系数 | 公司精算配置 | 0.65 ~ 1.35 |
| 渠道系数 | 渠道协议配置 | 0.85 ~ 1.15 |

#### FR-005 实时报价

- 用户填写完车辆信息后，3秒内返回报价结果
- 报价结果有效期：7天
- 支持不同方案对比展示

### 5.3 核保规则

#### FR-006 自动核保规则

| 规则编号 | 规则描述 | 触发条件 | 处理结果 |
|----------|----------|----------|----------|
| AU-001 | 新车购置价校验 | 车龄 ≤ 1年 | 偏差 > 20% 转人工 |
| AU-002 | 高龄车辆限制 | 车龄 > 10年 | 不承保车损险 |
| AU-003 | 高风险车型 | 品牌在黑名单中 | 转人工核保 |
| AU-004 | 短期投保限制 | 保险期间 < 1年 | 转人工核保 |
| AU-005 | 投保人信用评分 | 信用分 < 600 | 拒保 |
| AU-006 | 历史出险次数 | 近3年出险 ≥ 5次 | 转人工核保 |

#### FR-007 人工核保队列

- 按提交时间排序进入队列
- 支持按风险等级自动分级（高/中/低）
- 核保员处理时效：高风险 ≤ 2小时，中风险 ≤ 4小时，低风险 ≤ 8小时
- 超时自动升级提醒

### 5.4 支付与出单

#### FR-008 支付方式

| 支付方式 | 接口方 | 手续费 | 到账时效 |
|----------|--------|--------|----------|
| 微信支付 | 微信支付API | 0.6% | 实时 |
| 支付宝 | 支付宝API | 0.55% | 实时 |
| 银行卡快捷支付 | 银联在线 | 0.45% | 实时 |
| 对公转账 | 银行代扣 | 0 | T+1 |

#### FR-009 电子保单生成

- 支付成功后 30 秒内生成电子保单（PDF格式）
- 保单号生成规则：地区码(2) + 险种码(2) + 年份(4) + 序列号(10)
- 自动推送至客户APP/短信/邮箱
- 支持保单验真（二维码+防伪码）

## 6. 非功能需求

### 6.1 性能需求

| 指标 | 要求 |
|------|------|
| 报价接口响应时间 | P99 ≤ 3s |
| 投保提交响应时间 | P99 ≤ 5s |
| 并发用户数 | ≥ 5000 |
| 日处理投保量 | ≥ 50万笔 |
| 系统可用性 | 99.95% |

### 6.2 安全需求

- 所有API接口HTTPS加密传输
- 敏感字段（证件号、银行卡号）加密存储
- 操作日志全量记录，保留 ≥ 3年
- 符合《个人信息保护法》及银保监会数据安全要求
