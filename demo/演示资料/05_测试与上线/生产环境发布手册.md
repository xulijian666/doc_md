## 1. 发布流程

## 3. 发布步骤

### 3.1 标准发布流程

```
Step 1: 通知相关方（发布群公告）
    │
    ▼
Step 2: 暂停定时任务（如有）
    │
    ▼
Step 3: 数据库变更（如有）
    │
    ├── 执行DDL脚本
    │
    ├── 执行DML脚本
    │
    └── 验证数据正确性
    │
    ▼
Step 4: 应用发布
    │
    ├── 灰度发布（10%流量）
    │
    ├── 观察5分钟（日志+监控+业务指标）
    │
    ├── 扩至50%流量
    │
    ├── 观察5分钟
    │
    └── 全量发布
    │
    ▼
Step 5: 健康检查
    │
    ├── 服务注册状态检查
    │
    ├── 接口响应检查
    │
    └── 业务功能验证
    │
    ▼
Step 6: 恢复定时任务
    │
    ▼
Step 7: 通知发布完成
```

### 3.2 灰度策略

| 阶段 | 流量比例 | 观察时间 | 回滚条件 |
|------|----------|----------|----------|
| 灰度10% | 10% | 5分钟 | 错误率 > 1% |
| 灰度50% | 50% | 5分钟 | 错误率 > 0.5% |
| 全量 | 100% | 15分钟 | 错误率 > 0.1% |

### 3.3 发布命令

```bash
# 1. 构建镜像
docker build -t registry.insurance.com/insur-app:${VERSION} .
docker push registry.insurance.com/insur-app:${VERSION}

# 2. 更新K8s部署
kubectl set image deployment/insur-app \
  insur-app=registry.insurance.com/insur-app:${VERSION} \
  -n production

# 3. 查看滚动更新状态
kubectl rollout status deployment/insur-app -n production

# 4. 回滚命令（如需要）
kubectl rollout undo deployment/insur-app -n production
```

## 5. 回滚方案

### 5.1 回滚决策

| 触发条件 | 决策人 | 回滚方式 |
|----------|--------|----------|
| 错误率 > 1% | 值班开发 | 自动回滚 |
| 核心接口不可用 | 技术负责人 | 立即回滚 |
| 业务数据异常 | CTO | 立即回滚+数据修复 |
| 性能严重下降 | 技术负责人 | 回滚或扩容 |

### 5.2 回滚步骤

```
Step 1: 确认回滚决策
    │
    ▼
Step 2: 应用回滚
    │
    └── kubectl rollout undo deployment/xxx -n production
    │
    ▼
Step 3: 数据库回滚（如有DDL变更）
    │
    └── 执行回滚SQL脚本
    │
    ▼
Step 4: 验证回滚成功
    │
    ├── 服务状态检查
    │
    └── 业务功能验证
    │
    ▼
Step 5: 通知相关方
    │
    ▼
Step 6: 编写事故报告
```

### 5.3 数据库回滚脚本

```sql
-- 回滚脚本示例
-- 发布版本: V3.2
-- 回滚人: DBA
-- 回滚时间: 2026-05-XX

-- 1. 删除新增表
DROP TABLE IF EXISTS t_new_feature;

-- 2. 回滚字段变更
ALTER TABLE t_policy DROP COLUMN new_field;

-- 3. 回滚数据变更
UPDATE t_config SET value = 'old_value' WHERE key = 'feature_flag';
```

## 6. 应急预案

### 6.1 常见问题处理

| 问题 | 处理方式 |
|------|----------|
| 镜像拉取失败 | 检查镜像仓库，切换备用仓库 |
| 启动失败 | 检查配置、环境变量、依赖服务 |
| 内存溢出 | 回滚+调整JVM参数 |
| 数据库连接池耗尽 | 扩容连接池+检查慢SQL |
| 支付回调异常 | 检查支付网关+手动补偿 |
