## 1. 报表概述

### 1.1 报表名称

月度保费收入统计报表

### 1.2 报表目的

统计各产品线、各渠道、各地区的月度保费收入情况，为业务决策提供数据支撑。

### 1.3 使用对象

- 精算部：费率调整依据
- 销售部：渠道业绩评估
- 管理层：经营分析决策

## 2. 统计口径

### 2.1 保费统计口径

| 口径 | 定义 | SQL条件 |
|------|------|---------|
| 签单保费 | 当月签发保单的保费总额 | issue_date 在统计月内 |
| 实收保费 | 当月实际到账的保费总额 | payment.paid_time 在统计月内且status=SUCCESS |
| 批增保费 | 当月保全批改增加的保费 | baoku.change_type='INCREASE' |
| 批减保费 | 当月保全批改减少的保费 | baoku.change_type='DECREASE' |
| 退保保费 | 当月退保的保费总额 | surrender_time 在统计月内 |
| 满期保费 | 当月到期保单的保费总额 | expire_date 在统计月内 |

### 2.2 时间口径

- 统计周期：自然月（每月1日 00:00:00 至月末 23:59:59）
- 数据截止：T+1日 02:00 完成上月数据统计
- 数据锁定：T+3日锁定上月数据，不再更新

### 2.3 数据排除规则

| 排除项 | 说明 |
|--------|------|
| 测试保单 | product_code 以 'TEST' 开头 |
| 冲正保单 | status='REVERSED' |
| 作废保单 | status='VOID' 且 issue_date 在统计月内 |

## 3. 报表维度

| 维度 | 字段 | 说明 |
|------|------|------|
| 统计月份 | stat_month | yyyy-MM格式 |
| 产品线 | product_type | CAR/HEALTH/LIFE/PROPERTY |
| 产品 | product_code | 具体产品编码 |
| 渠道 | channel_code | 投保渠道 |
| 地区 | region_code | 省级行政区划代码 |
| 机构 | org_code | 分支机构编码 |

## 4. 报表指标

### 4.1 核心指标

| 指标名称 | 计算公式 | 单位 | 精度 |
|----------|----------|------|------|
| 签单保费 | SUM(premium) WHERE issue_date in month | 元 | 2位小数 |
| 签单件数 | COUNT(policy_no) WHERE issue_date in month | 件 | 整数 |
| 件均保费 | 签单保费 / 签单件数 | 元 | 2位小数 |
| 实收保费 | SUM(pay_amount) WHERE paid_time in month | 元 | 2位小数 |
| 实收率 | 实收保费 / 签单保费 | % | 2位小数 |
| 退保保费 | SUM(surrender_amount) WHERE surrender_time in month | 元 | 2位小数 |
| 退保率 | 退保保费 / 期初有效保单保费 | % | 2位小数 |
| 增长率 | (本月签单保费 - 上月签单保费) / 上月签单保费 | % | 2位小数 |
| 同比增长率 | (本月签单保费 - 去年同月签单保费) / 去年同月签单保费 | % | 2位小数 |

### 4.2 衍生指标

| 指标名称 | 计算公式 | 说明 |
|----------|----------|------|
| 有效保单数 | 状态为EFFECTIVE的保单总数 | 时点数据 |
| 有效保额 | SUM(insured_amount) WHERE status=EFFECTIVE | 风险暴露 |
| 赔付率 | 赔付金额 / 已赚保费 | 盈利能力 |
| 综合成本率 | (赔付金额+费用) / 已赚保费 | 经营效率 |

## 6. 报表生成流程

```
数据采集（TiDB ETL）
    │
    ▼
数据校验（完整性+一致性检查）
    │
    ├── 校验通过 ──▶ 指标计算 ──▶ 报表生成 ──▶ 发布
    │
    └── 校验失败 ──▶ 告警通知 ──▶ 人工排查 ──▶ 修复重算
```
