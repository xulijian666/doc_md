## 1. 数据分类分级

### 1.1 数据分级标准

| 级别 | 定义 | 示例 | 保护要求 |
|------|------|------|----------|
| L1-公开 | 可对外公开的数据 | 产品介绍、公告 | 基本保护 |
| L2-内部 | 仅限公司内部使用 | 业务报表、统计 | 访问控制 |
| L3-敏感 | 泄露可能造成损害 | 客户姓名、手机号 | 加密+脱敏 |
| L4-高度敏感 | 泄露将造成严重损害 | 证件号、银行卡号、健康信息 | 强加密+审计 |

### 1.2 保险业务数据分类

| 数据类别 | 分级 | 包含字段 |
|----------|------|----------|
| 身份信息 | L4 | 身份证号、护照号、人脸信息 |
| 联系信息 | L3 | 手机号、邮箱、地址 |
| 财务信息 | L4 | 银行卡号、收入信息、保费 |
| 健康信息 | L4 | 病历、诊断、体检报告 |
| 保单信息 | L3 | 保单号、险种、保额 |
| 理赔信息 | L4 | 理赔材料、赔付金额 |
| 行为信息 | L2 | 浏览记录、操作日志 |

## 2. 数据采集规范

### 2.1 最小必要原则

| 业务场景 | 允许采集 | 禁止采集 |
|----------|----------|----------|
| 车险投保 | 车牌号、VIN、姓名、证件号、手机号 | 无关家庭成员信息 |
| 健康险投保 | 健康告知问卷 | 完整病历档案 |
| 理赔 | 诊断证明、费用发票 | 非相关科室病历 |
| 保全 | 变更所需字段 | 未变更字段的更新采集 |

### 2.2 知情同意

| 场景 | 同意方式 | 保留要求 |
|------|----------|----------|
| 投保 | 电子签名确认 | 合同期满后5年 |
| 理赔 | 勾选同意+短信验证 | 案件结案后10年 |
| 数据共享 | 单独同意 | 共享结束后5年 |
| 敏感信息 | 单独同意+显著提示 | 合同期满后5年 |

## 3. 数据存储安全

### 3.1 加密方案

| 数据类型 | 加密算法 | 密钥管理 |
|----------|----------|----------|
| 证件号码 | AES-256-GCM | HSM硬件加密机 |
| 银行卡号 | AES-256-GCM | HSM硬件加密机 |
| 手机号 | AES-256-GCM | HSM硬件加密机 |
| 人脸信息 | SHA-256哈希 | 不可逆存储 |
| 传输通道 | TLS 1.2+ | 证书管理平台 |

### 3.2 脱敏规则

| 字段 | 脱敏规则 | 脱敏后示例 |
|------|----------|-----------|
| 身份证号 | 保留前3后4 | 110***1234 |
| 手机号 | 保留前3后4 | 138****8000 |
| 银行卡号 | 保留前4后4 | 6222****7890 |
| 姓名 | 保留姓 | 张** |
| 地址 | 保留省市 | 北京市朝阳区*** |
| 邮箱 | 保留首字母和域名 | z***@example.com |

### 3.3 访问控制

| 角色 | 可访问数据级别 | 审批要求 |
|------|---------------|----------|
| 客服坐席 | L3（脱敏） | 系统自动授权 |
| 核保员 | L3+L4（脱敏） | 主管审批 |
| 理赔员 | L3+L4（部分明文） | 主管审批 |
| 管理层 | L3+L4（报表脱敏） | 系统自动授权 |
| 开发人员 | L2（测试环境脱敏数据） | CTO审批 |
| 外部审计 | L3（脱敏） | 合规部审批 |

## 4. 数据使用规范

### 4.1 使用场景管控

| 使用场景 | 允许 | 条件 |
|----------|------|------|
| 业务处理 | 是 | 遵循最小必要 |
| 数据分析 | 是 | 使用脱敏数据 |
| 模型训练 | 是 | 数据匿名化处理 |
| 对外共享 | 需审批 | 单独同意+合同约束 |
| 跨境传输 | 限制 | 安全评估+监管审批 |
| 营销推送 | 需同意 | 用户可退订 |

### 4.2 数据脱敏展示

系统各页面展示个人信息时，必须遵循以下规则：

```
身份证号：110***********1234   （前3后4）
银行卡号：6222 **** **** 7890  （前4后4）
手机号：  138****8000          （前3后4）
姓名：    张*                  （保留姓）
```

## 5. 数据共享与委托处理

### 5.1 共享审批流程

```
业务部门提出共享需求
    │
    ▼
数据保护官 (DPO) 评估
    │
    ▼
个人信息影响评估 (PIA)
    │
    ├── 低风险 ──▶ DPO审批 ──▶ 签署数据处理协议 ──▶ 执行共享
    │
    └── 高风险 ──▶ 管理层审批 ──▶ 监管报备 ──▶ 签署协议 ──▶ 执行共享
```

### 5.2 第三方管理

| 要求 | 说明 |
|------|------|
| 资质审查 | 数据安全能力评估 |
| 合同约束 | 数据处理协议 (DPA) |
| 范围限制 | 仅共享必要数据 |
| 安全审计 | 年度安全评估 |
| 退出机制 | 合同终止后数据删除确认 |

## 6. 数据主体权利

### 6.1 权利清单

| 权利 | 说明 | 响应时限 |
|------|------|----------|
| 知情权 | 了解数据处理方式 | 即时（隐私政策） |
| 查询权 | 查询个人信息 | 15个工作日 |
| 更正权 | 更正错误信息 | 15个工作日 |
| 删除权 | 删除个人信息 | 15个工作日 |
| 撤回同意 | 撤回数据处理同意 | 即时生效 |
| 可携带权 | 获取数据副本 | 15个工作日 |
| 投诉权 | 向监管部门投诉 | 提供投诉渠道 |

### 6.2 系统实现

- 客户APP提供个人信息管理入口
- 支持在线查看、导出、申请删除
- 删除请求自动触发数据清理流程
- 删除操作日志记录（证明已执行）

## 7. 安全事件响应

### 7.1 事件分级

| 级别 | 定义 | 响应时间 | 报告对象 |
|------|------|----------|----------|
| 一级 | 泄露 > 100万条 | 1小时内 | 监管部门+管理层 |
| 二级 | 泄露 10-100万条 | 4小时内 | 管理层 |
| 三级 | 泄露 < 10万条 | 24小时内 | 合规部 |
| 四级 | 疑似泄露 | 48小时内 | 安全部 |

### 7.2 响应流程

```
事件发现
    │
    ▼
初步评估（1小时内）
    │
    ▼
启动应急响应
    │
    ├── 止损措施（隔离、封锁）
    │
    ├── 影响评估（范围、数量）
    │
    ├── 监管报告（按级别时限）
    │
    ├── 用户通知（受影响用户）
    │
    └── 整改措施（漏洞修复）
```
