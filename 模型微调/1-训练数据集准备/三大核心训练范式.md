# 大模型微调三大核心训练范式

## 总述

CPT、SFT、DPO 属于"下游增量训练"阶段，前提都是必须有一个已经完全训练好的**基座模型（Base Model）**。

**最佳实践链路：** CPT（补知识，可选）→ SFT（学做事，必选）→ DPO（做得更好，可选）

---

## 阶段一：基座模型的诞生（前置预训练 Pre-training）

- **做法：** 喂给模型海量无标注文本（互联网数据、维基百科、图书等，动辄几万亿 Token），让模型做"连词成句"的填空题
- **结果：** 诞生基座模型（如 Qwen-7B-Base），拥有强大的语言理解能力和常识，但还不会听人话、不会好好回答问题

---

## 阶段二：增量调优（三大核心范式）

### 1. CPT（继续预训练 - Continue Pre-Training）

| 维度 | 说明 |
|------|------|
| **目的** | 给模型注入特定行业/领域专业知识 |
| **通俗比喻** | 让大学生去读研、读博 |
| **数据格式** | 纯文本（Raw Text），无需人工标注 |
| **数据来源** | 专业 PDF、文档、电子书等（几 GB 级别） |
| **训练方式** | 继续玩"文字接龙"，在潜意识里建立底层专业词汇和知识地图 |
| **市面地位** | 行业专属大模型的敲门砖 |
| **典型场景** | 金融法规、保险精算、医疗病历、公司内部文档等垂直领域知识注入 |
| **调优效果** | 模型补齐了行业专业知识，建立了专业术语和知识体系 |

### 2. SFT（有监督微调 - Supervised Fine-Tuning）

| 维度 | 说明 |
|------|------|
| **目的** | 教模型听懂指令、按规范格式回答问题 |
| **通俗比喻** | 教大学生学会职场礼仪和规范工作 |
| **数据格式** | 提示词-回复对（Prompt-Response），如 `{"prompt": "...", "response": "..."}` |
| **数据来源** | 成千上万条高质量的人工编写正确示范 |
| **训练方式** | 通过问答对手把手教模型学会对话交互、角色扮演、格式输出（如 JSON） |
| **市面地位** | 使用率 100% 的绝对核心，只要是微调就离不开 SFT |
| **支持方式** | 全参数微调（效果好但贵）和高效微调（如 LoRA，省时省钱） |
| **典型场景** | 训练客服机器人、规范代码生成格式、让模型学会使用特定 Agent 工具 |
| **调优效果** | 模型从只会"顺着字往下编"变成能听懂 Prompt 指令并以标准格式回答 |

### 3. DPO（直接偏好优化 - Direct Preference Optimization）

| 维度 | 说明 |
|------|------|
| **目的** | 降低幻觉、调整语气，让输出更符合人类偏好和安全合规要求 |
| **通俗比喻** | 给员工发奖金或扣绩效，修正他的价值观 |
| **数据格式** | 三元组（Prompt + Positive 好的回答 + Negative 坏的回答） |
| **数据来源** | 搜集业务中的 Bad Case（坏用例）作为负反馈 |
| **训练方式** | 直接在数学上通过对比"好"与"坏"，拉高优质回答概率，压低敷衍/错误/有毒回答概率 |
| **市面地位** | 目前最火、最主流的对齐技术，正在迅速取代老一代 PPO（RLHF） |
| **前置条件** | 必须建立在 SFT 之后的模型之上 |
| **典型场景** | 降低幻觉、调整语气、确保输出安全合规、让回答更简明扼要 |
| **调优效果** | 模型不仅"能干活"，还能"把活干得漂亮" |

### 补充：GRPO（群体相对策略优化）

针对数学、代码等"答案高度可验证"的逻辑推理任务，市面上开始流行 GRPO 等新型强化学习方法。但在通用的企业落地场景中，CPT、SFT、DPO 依然是最稳固的主力军。

---

## 2026年行业微调落地现状

### 组合一：常规任务（80% 场景）—— 直接 SFT
- 基座模型通用能力已足够，只需特定语气或固定格式
- 准备几千条高质量 SFT 数据即可上线

### 组合二：垂直领域（金融、法律、医疗）—— CPT → SFT
- 模型必须先通过 CPT 吃透行业法条、术语
- 再通过 SFT 训练如何做咨询和解答

### 组合三：极致体验（C端产品、降幻觉）—— SFT → DPO
- 模型已有专业能力，但需防止说错话
- 搜集 Bad Case 作为负反馈，跑一轮 DPO 对齐

---

## 三大范式对比总结

| 范式 | 数据类型 | 是否需要标注 | 核心作用 | 是否必选 |
|------|----------|-------------|----------|----------|
| **CPT** | 纯文本 | 否 | 补专业知识 | 可选 |
| **SFT** | Prompt-Response 对 | 是 | 学会听指令、规范输出 | 必选 |
| **DPO** | Prompt + 好/坏回答三元组 | 是 | 降低幻觉、对齐人类偏好 | 可选 |
