整体流程

Step 1 · 文档抽取
.docx — 通过 Pandoc 将 Word 文档转为 Markdown，随后进行后处理：标题自动编号、OLE 嵌入对象提取、图片格式转换（BMP → PNG）、Obsidian Wikilink 格式化、格式清洗规整
.xlsx / .xls — 使用 openpyxl 提取单元格数据，生成 Markdown 表格
.pdf — 调用 MarkItDown 抽取纯文本内容
图片 — 通过 Pillow 将 OLE 对象中提取的原始 BMP 字节流无损转换为 PNG 图片
Step 2 · 本体映射
将抽取的结构化 Markdown 内容映射到本体模型（方案待定，需跟随本体设计方案）
Step 3 · 文档知识管理
基于 Obsidian 双链（Bidirectional Linking）实现文档间的双向关联，构建可浏览、可检索的知识网络

开源技术方案

开源项目 GitHub 地址 说明
Pandoc github.com/jgm/pandoc 开源文档格式转换的"瑞士军刀"，负责通过命令行将 .docx 文本转为 Markdown
MarkItDown github.com/microsoft/markitdown 微软开源的多格式转 Markdown 工具，专门用来抽取 .pdf 文件的纯文本
Pillow github.com/python-pillow/Pillow 开源图像处理库，负责将从 OLE 对象中提取的原始 BMP 字节流无损转换为 PNG 图片
